# تقنيات سوفت - نظام مطعم متكامل

## التشغيل المحلي
```
npm install
npm start
```

## بناء ملف EXE
```
npm run dist
```
أو ادفع (push) إلى الفرع `main` على GitHub وسيقوم `GitHub Actions` ببناء الملف تلقائياً
ورفعه كـ Artifact باسم `Technologies-Soft-Installer`.

## بيانات الدخول الافتراضية (أول تشغيل فقط)
| الدور | اسم المستخدم | كلمة المرور |
|---|---|---|
| مدير عام | admin | Admin@2026 |
| محاسب | accountant | Acc@2026 |
| كاشير | cashier | Cash@2026 |

⚠️ **يجب تغيير كلمات المرور هذه فوراً بعد أول تسجيل دخول.**
البرنامج سيطلب منك ذلك تلقائياً، ويمكن أيضاً تغييرها من: الإعدادات → تغيير كلمة المرور.

عند أول تشغيل، يُكتب ملف نصي يحتوي هذه البيانات داخل مجلد بيانات المستخدم
(Roaming/تقنيات سوفت...) باسم `بيانات_الدخول_الافتراضية.txt` — يُفضّل حذفه بعد
تغيير كلمات المرور.

## ⚠️ تنبيه أمان قبل النشر العام
هذا الريبو يحتوي منطق أعمال كامل لنظام نقاط بيع. إن كنت تنشره على GitHub:
- اجعل الريبو **Private** إن كان مخصصاً لعميل واحد.
- لا ترفع أبداً ملفات `.db` الفعلية الموجودة في مجلد بيانات المستخدم.
- غيّر `appId` في `package.json` إذا كنت تبيع النظام لعملاء متعددين تحت أسماء مختلفة.

## ملاحظات فنية
- قاعدة البيانات: SQLite عبر `better-sqlite3`، تُخزَّن في مجلد بيانات المستخدم الخاص بنظام التشغيل.
- الطباعة الحرارية: تُحاول الاتصال بطابعة USB تلقائياً؛ إن فشل، تفتح نافذة طباعة عادية كحل احتياطي.
  لتحديد طابعة شبكية، عيّن متغير البيئة `THERMAL_PRINTER_INTERFACE` (مثلاً `tcp://192.168.1.50`).
- الأمان: تم تفعيل `contextIsolation` وإلغاء `nodeIntegration` في صفحة الواجهة؛ كل الاتصال
  بقاعدة البيانات يمر عبر `preload.js` وقائمة قنوات مسموحة فقط.
